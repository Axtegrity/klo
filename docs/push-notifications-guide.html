Overview

What push notifications are and how they work in KLO

Your KLO app supports push notifications across web browsers, iOS, and Android. Users opt in from their Profile Settings, and you can send notifications from the Admin Dashboard.

Push notifications are messages that appear on a user's device even when they're not actively using your app — banners on phones, pop-ups on desktop. Users must grant permission before receiving them.

Method 1: Admin Dashboard

Send from your computer at keithlodom.ai/admin

Desktop / Laptop

Getting There

Open your browser, sign in as admin, and navigate to the Notifications tab.

  1. Sign in at keithlodom.ai

    Use your admin credentials (owner or admin role required)

  2. Go to /admin

    Click "Admin" in the navigation or go directly to keithlodom.ai/admin

  3. Click the "Notifications" tab

    You'll see subscriber stats, the compose form, and a list of all subscribers

Sending a Broadcast (All Users)

  1. Select "Broadcast to All"

    This is the default. Every subscribed user on every device will receive it.

  2. Fill in the notification

    Title: Short & punchy, under 50 characters (bold headline on the device)

    Body: 1-2 sentences. Be clear and actionable.

    Link URL: Where they go when they tap. Use paths like /vault, /assessments, /booking

  3. Click "Send Notification"

    A green banner confirms how many devices received it.

Method 1: Admin Dashboard (cont.)

Targeting a specific user instead of everyone

Sending to a Specific User

Use this when a notification is relevant to only one person.

  1. Click "Specific User" (instead of "Broadcast to All")

    A new "User ID" field will appear in the form.

  2. Find the User ID

    Switch to the Users tab in the admin dashboard. Each user row has a UUID — copy it.

  3. Paste the User ID, compose your message, and send

    Only that user's devices will receive the notification.

Method 2: From the Mobile App

Send push notifications directly from your iPhone or Android

iOS / Android

Sending from Your Phone

The admin dashboard works in the KLO mobile app too — same features, optimized for touch.

  1. Open the KLO app on your phone

    Make sure you're signed in with your admin account.

  2. Navigate to Admin → Notifications tab

    The admin dashboard is fully responsive. Tap the "Notifications" tab just like on desktop.

  3. Choose broadcast or specific user

    Tap "Broadcast to All" or "Specific User" — same options as desktop.

  4. Fill in Title, Body, and Link URL

    Compose your notification. Keep the title short since you're typing on mobile.

  5. Tap "Send Notification"

    You'll see the green confirmation with delivery count.

Pro Tip: Quick Sends on the Go This is perfect for time-sensitive notifications like "Session starting in 10 minutes!" or "New framework just dropped" — no need to get to a computer. The mobile app uses the same API, so everything works identically.
Who Can Send from Mobile? Only users with owner or admin roles can access the Notifications tab and send push notifications. Regular users won't see it.

How Users Subscribe

What your users need to do to start receiving notifications

Web Browser

Subscribing on Desktop / Laptop

Works in Chrome, Edge, Firefox, and Safari 16.4+

  1. Go to Profile → Settings

    The "Push Notifications" toggle is under Notification Preferences.

  2. Toggle on Push Notifications

    The browser shows a permission prompt: "keithlodom.ai wants to send notifications."

  3. Click "Allow"

    Done! The toggle turns blue. They'll now receive push notifications in their browser.

iOS / Android App

Subscribing on Mobile

The app handles most of this automatically

  1. App prompts on first launch

    When a user first opens the KLO app, it automatically asks for push notification permission. If they tap "Allow", they're subscribed immediately.

  2. Can also toggle in Profile → Settings

    Same toggle as web. If they declined the initial prompt, they can enable it here (the app will re-prompt for permission).

If a User Blocked Notifications If someone previously tapped "Don't Allow" (iOS) or "Block" (browser), they need to fix it in their device settings:
iPhone: Settings → KLO → Notifications → Allow Notifications
Chrome: Click lock icon in address bar → Site Settings → Notifications → Allow

Unsubscribing & Best Practices

How users opt out, and how to keep engagement high

Unsubscribing

Users can toggle off push notifications from their Profile → Settings at any time. This immediately removes their subscription from the server. They stop receiving notifications on that device right away.

Best Practices

Do

  • ✓  Keep titles under 50 characters
  • ✓  Make the body actionable
  • ✓  Always include a link URL
  • ✓  Send 1-3x per week max
  • ✓  Send during business hours (Tue-Thu is best)
  • ✓  Target specific users when relevant

Don't

  • ✗  Send more than once a day
  • ✗  Use ALL CAPS or clickbait
  • ✗  Send without a clear purpose
  • ✗  Send late at night
  • ✗  Send the same message twice
  • ✗  Forget to test on your own device first

Technical Reference

For developers and technical team members

API Endpoints

Endpoint Method Auth Purpose
/api/push/subscribe POST User Register a push subscription
/api/push/unsubscribe DELETE User Remove a push subscription
/api/push/send POST Admin Send notification (broadcast or targeted)
/api/push/subscribers GET Admin List all subscribers with stats

Send API — Request Examples

// Broadcast to all subscribers

{
  "title": "Your Report is Ready",
  "body": "View your AI readiness report.",
  "userId": "user-uuid-here"
}

// Send to multiple specific users
{
  "title": "Pro Members Only",
  "body": "Exclusive content just dropped.",
  "userIds": ["uuid-1", "uuid-2"]
}
